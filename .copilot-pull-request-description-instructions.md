# Hướng dẫn tạo Pull Request Description

Tất cả tiêu đề và nội dung Pull Request (PR) phải được viết bằng **tiếng Việt** có dấu.

## Quy tắc:

1. **Tiêu đề PR**: `<type>: <mô tả ngắn gọn>`

2. **Type** (giữ nguyên tiếng Anh):
   - `feat`: Tính năng mới
   - `fix`: Sửa lỗi
   - `refactor`: Tái cấu trúc code
   - `docs`: Cập nhật tài liệu
   - `style`: Định dạng code (không ảnh hưởng logic)
   - `test`: Thêm/sửa test
   - `chore`: Cập nhật build, dependencies

3. **Mô tả ngắn gọn** (Tiêu đề):
   - Viết bằng tiếng Việt có dấu
   - Ngắn gọn, rõ ràng
   - Bắt đầu bằng động từ: "Bổ sung", "Cập nhật", "Sửa", "Xóa", "Thêm", "Cải thiện", "Tối ưu"

4. **Nội dung chi tiết** (Body):
   - **Tổng quan**: Mô tả ngắn gọn về những gì PR này thực hiện.
   - **Chi tiết thay đổi**:
     - Liệt kê các thay đổi chính dưới dạng danh sách (bullet points).
     - Giải thích **tại sao** thay đổi này cần thiết.
     - Mô tả **những gì** đã thay đổi về mặt kỹ thuật (nếu cần).
   - **Ảnh hưởng**: Liệt kê các module hoặc chức năng bị ảnh hưởng (nếu có).
   - **Ghi chú**: Các lưu ý đặc biệt cho người review hoặc tester.

## Ví dụ:

**Tiêu đề:**
`feat: Bổ sung chức năng đăng nhập VBDLIS tự động`

**Nội dung:**
### Tổng quan
PR này bổ sung khả năng đăng nhập tự động vào hệ thống VBDLIS để hỗ trợ các tác vụ crawl dữ liệu.

### Chi tiết thay đổi
- Thêm `VbdlisAuthService` để xử lý logic đăng nhập và logout.
- Cập nhật `SessionManager` để lưu trữ token xác thực.
- Thêm cơ chế tự động refresh token khi hết hạn.
- Cập nhật UI trang đăng nhập để hiển thị trạng thái kết nối.

### Ảnh hưởng
- Module: `Haihv.Vbdlis.Tools.Desktop`
- Các chức năng liên quan đến xác thực người dùng.

### Ghi chú
- Cần cấu hình thông tin tài khoản trong file config trước khi chạy test.
